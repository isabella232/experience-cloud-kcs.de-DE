# Finden Sie heraus, welche Anforderungen AEM Dispatcher-Cache

## Beschreibung {#description}

<br>Problem<br><br>
Es muss bestimmt werden, warum HTTP-Anforderungen/-Antworten vom AEM Dispatcher nicht zwischengespeichert werden.
<br><br>Umgebung<br><br>
AEM Dispatcher 4.3.3


## Auflösung {#resolution}


Die Dokumentation behandelt die wichtigsten Szenarien, die sich auf die Cache-Fähigkeit von Anforderungen und Antworten auswirken, die durch das AEM Dispatcher-Modul ausgeführt werden. Alle Cache-Regeln werden nicht abgedeckt.

<b>HTTP-Anforderungs- und Caching-Regeln</b>

Damit eine HTTP-Anforderung vom Dispatcher zwischengespeichert werden kann, muss sie den folgenden Regeln entsprechen:

1. URL ist absolut (funktioniert nicht mit . oder ..) eine Dateierweiterung hat und der Dateiname in der URL als Datei im Dateisystem funktioniert.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   URI not canonical: %s.    
   Unable to map URI to file: %s.    
   cache file path too long    
   temporary file path too long    
   request URL has no extension
   ```



   <b>Beispiele:</b>


   | `/content/test` | nicht zwischengespeichert |
   | --- | --- |
   | `/content/test.html` | zwischengespeichert |


2. Nach der ersten Dateierweiterung gibt es keine Schrägstriche.<br>

   <b>Verwandte Protokollmeldungen:</b>


   ```
   request URL has trailing slash
   ```



   <b>Beispiele:</b>


   | `/content/test.html/` | nicht zwischengespeichert |
   | --- | --- |
   | `/content/test.html/foo.jpg` | nicht zwischengespeichert |
   | `/content/test.html` | zwischengespeichert |


3. Es verwendet HTTP-GET- oder HEAD-Methode.<br>

   <b>Verwandte Protokollmeldungen:</b>


   ```
   request method is neither GET nor HEAD
   ```



   <b>Beispiele:</b>


   | `HEAD /content/test.html HTTP/1.1` | zwischengespeichert |
   | --- | --- |
   | `GET /content/dam/test.jpg HTTP/1.1` | nicht zwischengespeichert |
   | `POST /content/test.html HTTP/1.1` | nicht zwischengespeichert |


4. Die Anfrage wird von /CACHE = /RULES verweigert, das im Dispatcher definiert ist. JEDE FARM-KONFIGURATION.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   URI not  in cache rules: %s request URL not in cache rules
   ```



   <b>Beispiele:</b>


   | <br><br>    ```<br>    Farm has /cache = /rules and a request is received for /content/test.html<br>        /cache<br>        {<br>         ...<br>         /rules<br>         {<br>          /0001 {<br>          type "deny"<br>          glob "*"<br>          }<br>         }<br>        }<br>    ```<br><br> | nicht zwischengespeichert |
   | --- | --- |
   | <br><br>    ```<br>    Farm has /cache = /rules and a request is received for /content/test.html<br>        /cache<br>        {<br>         ...<br>         /rules<br>         {<br>          /0001 {<br>          type "allow"<br>          glob "*"<br>          }<br>         }<br>        }<br>    ```<br><br> | zwischengespeichert |


5. Die HTTP-Anfrage enthält keinen Autorisierungs-Header oder enthält den Header, aber /ALLOWAUTHORIZED ist in der FARM-KONFIGURATION auf 1 gesetzt.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   request contains authorization
   ```



   <b>Beispiele:</b>


   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | nicht zwischengespeichert |
   | --- | --- |
   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Authorization: Basic YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “1”    }<br>    ```<br><br> | zwischengespeichert |


6. Die HTTP-Anfrage enthält LOGIN-TOKEN oder AUTHORIZATION-COOKIE im Cookie-Header oder enthält entweder (oder beide) dieser Cookies und /ALLOWAUTHORIZED ist in der FARM-KONFIGURATION auf 1 gesetzt.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   request contains authorization
   ```



   <b>Beispiele:</b>


   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: login-token=…<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | nicht zwischengespeichert |
   | --- | --- |
   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: authorization=YWxhZGRpbjpvcGVuc2VzYW1l<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “0”    }<br>    ```<br><br> | nicht zwischengespeichert |
   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html HTTP/1.1    Cookie: login-token=…<br>        dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /cache {<br>           /allowAuthorized “1”    }<br>    ```<br><br> | zwischengespeichert |


7. Die URL enthält einen QUERYSTRING -Parameter und der Parameter darf über die KONFIGURATION /IGNOREURLPARAMS DER FARM ignoriert werden.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   request contains a query string
   ```



   <b>Beispiele:</b>


   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    Farm /cache configuration:<br>        /ignoreUrlParams {     /0001 { /type “allow” /glob “*” }    }<br>    ```<br><br> | zwischengespeichert |
   | --- | --- |
   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    Farm /cache configuration:<br>        /ignoreUrlParams {     /0001 { /type “deny” /glob “*” } <br>           /0001 { /type “allow” /glob “test” }    }<br>    ```<br><br> | zwischengespeichert |
   | HTTP-Anforderung:<br><br>    ```<br>    GET /content/test.html?test=1 HTTP/1.1    dispatcher.any farm /cache = /allowAuthorized configuration:<br>        /ignoreUrlParams {<br>         /0001 { /type “deny” /glob “*” } <br>           /0001 { /type “allow” /glob “q” }    }<br>    ```<br><br> | nicht zwischengespeichert |



<b>HTTP-Antwort- und Caching-Regeln:</b>

Die von AEM zurückgegebene HTTP-Antwort kann zwischengespeichert werden, wenn die folgenden Kriterien erfüllt sind:

1. Der Dispatcher kann eine 200 OK-Antwort von einem der definierten &quot;RENDERS&quot;senden und empfangen.

   <b>Hinweis:</b> Wenn nicht [/timeout](https://experienceleague.adobe.com/docs/experience-manager-dispatcher/using/configuring/dispatcher-configuration.html?lang=de) auf /renders festgelegt ist oder auf 0 gesetzt ist, würde es ewig auf eine Verbindung zur AEM-Instanz warten, selbst wenn die Instanz ausfällt.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   Unable to send request to remote server. Unable to receive response from remote server. Remote server returned: %s No backend available.
   ```



   <b>Beispiele:</b>


   | HTTP-Anforderung:<br><br>    200 OK | zwischengespeichert |
   | --- | --- |
   | HTTP-Anforderung:<br><br>    404 Seite nicht gefunden | nicht zwischengespeichert |


2. Keine dieser Antwortheader ist in der Antwort vorhanden:

   - `Dispatcher: no-cache`
   - `Cache-control: no-cache`
   - `Pragma: no-cache`

   <b>Verwandte Protokollmeldungen:</b>


   ```
   Backend forbids caching: %s, sent: %s" response contains no_cache
   ```



   <b>Beispiele:</b>


   | HTTP-Antwort:<br><br>    200 OK<br><br>    Dispatcher: no-cache | nicht zwischengespeichert |
   | --- | --- |
   | HTTP-Antwort:<br>    <br><br>    200 OK<br><br>    Cache-Control: no-cache | nicht zwischengespeichert |
   | HTTP-Antwort:<br><br>    200 OK<br><br>    Pragma: no-cache | nicht zwischengespeichert |
   | HTTP-Antwort:<br><br>    200 OK<br> | zwischengespeichert |


3. Der Wert der Inhaltslänge ist größer als NULL-BYTES.
   <b>Verwandte Protokollmeldungen:</b>


   ```
   response content length is zero
   ```



   <b>Beispiele:</b>


   | HTTP-Antwort:<br><br>    200 OK<br><br>    Content-Length: 0 | nicht zwischengespeichert |
   | --- | --- |
   | HTTP-Antwort:<br><br>    200 OK<br><br>    Content-Length: 3120 | zwischengespeichert |
