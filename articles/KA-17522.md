# Einrichten der Adobe I/O: Authentifizierung - Schritt für Schritt

## Beschreibung

<br>In diesem Dokument erfahren Sie, wie Sie die Zertifikate, Schlüssel und Token erstellen, die ausgeführt werden können.<br><br><br><br><br>Ziel<br><br><br><br> <br><br>
Um klare, reproduzierbare Schritte zum Generieren eines Adobe I/O Bearer-Tokens für die Ausführung von API-Aufrufen für DMA-Lösungen wie [!DNL Target].


## Auflösung

<br><br>Schritte<br><br><br><br> <br><br>
<b>Schritte zum Erstellen eines Trägertokens zum Ausführen von Adobe I/O-API-Aufrufen:</b>

1. Privaten Schlüssel und öffentliches Zertifikat generieren
2. Integration in der Adobe I/O Console erstellen
3. JWT-Token generieren
4. JWT-Token für Zugriffs-Trägertoken austauschen
5. Benutzer-Zugriffs-Trägertoken zum Ausführen von API-Aufrufen


- <b>Generieren Sie einen privaten Schlüssel und ein öffentliches Zertifikat gemäß der folgenden Dokumentation:</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Auf einem Mac sind die folgenden Befehle im Terminal integriert. 

   Auf einem Computer müssen Sie [!DNL Cygwin] (oder ein anderes Tool mit persönlicher Präferenz) und führen Sie es über die Befehlszeile aus. Im Folgenden finden Sie die Schritte zum Herunterladen und Installieren [!DNL Cygwin]:

   1. Navigieren Sie zu https://cygwin.com/install.html
   2. Herunterladen und ausführen `setup-x86_64.exe`

   <b>*Hinweis</b>:  Ihr Basisverzeichnis lautet: `C:\cygwin64\home\WINDOWSUSER`

   Sie können während der Installationsphase nach zusätzlichen Paketen suchen und diese installieren. Ich empfehle, alles zu installieren, was mit &quot;`curl`&quot; und &quot;`ssh.`&quot;*


   Hier finden Sie den auszuführenden Befehl:
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Hinweis&lt;/b>:  Es werden mehrere Fragen gestellt, die Sie zum Generieren des Zertifikats ausfüllen müssen. Siehe Screenshot unten:
![rtaImage](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_0/step_par/image/rtaImage.png "rtaImage")<br> 

    Nachdem die Dateien erstellt wurden, konvertieren Sie den &quot;secret.key&quot;mithilfe des folgenden Befehls in &quot;secret.pem&quot;:
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -notify PEM -outform DER -` `in` `secret.pem -nocrypt secret.key` |
    | — |
