# Einrichten der Adobe IO: Authentifizierung - Schritt für Schritt

## Beschreibung

<br>In diesem Dokument erfahren Sie, wie Sie die Zertifikate, Schlüssel und Token erstellen, die ausgeführt werden können.<br><br><br><br><br>Ziel<br><br><br><br> <br><br>
Um klare, reproduzierbare Schritte zum Generieren eines Adobe IO-Trägertokens für die Ausführung von API-Aufrufen für DMA-Lösungen wie Target bereitzustellen.


## Auflösung

<br><br>Schritte<br><br><br><br> <br><br>
<b>Schritte zum Erstellen eines Trägertokens zum Ausführen von Adobe IO API-Aufrufen:</b>

1. Privaten Schlüssel und öffentliches Zertifikat generieren
2. Integration in der Adobe IO-Konsole erstellen
3. JWT-Token generieren
4. JWT-Token für Zugriffs-Trägertoken austauschen
5. Benutzer-Zugriffs-Trägertoken zum Ausführen von API-Aufrufen


- <b>Generieren Sie einen privaten Schlüssel und ein öffentliches Zertifikat gemäß der folgenden Dokumentation:</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Auf einem Mac sind die folgenden Befehle im Terminal integriert. 

   Auf einem Computer müssen Sie Cygwin (oder ein anderes Tool Ihrer Wahl) herunterladen und von der Befehlszeile ausführen. Im Folgenden finden Sie die Schritte zum Herunterladen und Installieren von Cygwin:

   1. Navigieren Sie zu https://cygwin.com/install.html
   2. Herunterladen und Ausführen von setup-x86_64.exe

   <b>*Hinweis</b>:  Ihr Basisverzeichnis lautet: C:\cygwin64\home\WINDOWSUSER

   Sie können während der Installationsphase nach zusätzlichen Paketen suchen und diese installieren. Ich empfehle, alles zu installieren, was mit &quot;curl&quot;und &quot;ssh&quot;zu tun hat.*


   Hier finden Sie den auszuführenden Befehl:
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Hinweis&lt;/b>:  Es werden mehrere Fragen gestellt, die Sie zum Generieren des Zertifikats ausfüllen müssen. Siehe Screenshot unten:

    Nachdem die Dateien erstellt wurden, konvertieren Sie die Datei secret.key mithilfe des folgenden Befehls in secret.pem:
    
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -notify PEM -outform DER -&quot;in` `secret.pem -nocrypt secret.key&quot; |
