# Einrichten von Adobe I/O: Authentifizierung – Schritt für Schritt

## Beschreibung

<br>In diesem Dokument erfahren Sie, wie Sie die Zertifikate, Schlüssel und Token erstellen, die ausgeführt werden können.<br><br><br><br><br>Ziel<br><br><br><br> <br><br>
Die Bereitstellung von klaren, reproduzierbaren Schritten zum Generieren eines Adobe I/O Bearer-Tokens für die Ausführung von API-Aufrufen für DMA-Lösungen wie [!DNL Target].


## Lösung

<br><br>Schritte<br><br><br><br> <br><br>
<b>Schritte zum Erstellen eines Bearer-Tokens zum Ausführen von Adobe I/O-API-Aufrufen:</b>

1. Generieren eines privaten Schlüsses und eines öffentlichen Zertifikats
2. Erstellen der Integration in der Adobe I/O Console
3. Generieren des JWT-Tokens
4. Austausch des JWT-Tokens gegen ein Zugriffs-Bearer-Token
5. Benutzer-Zugriffs-Bearer-Token zum Ausführen von API-Aufrufen


- <b>Generieren Sie einen privaten Schlüssel und ein öffentliches Zertifikat gemäß der folgenden Dokumentation:</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Auf einem Mac sind die folgenden Befehle im Terminal integriert. 

   Auf einem Computer müssen Sie [!DNL Cygwin] (oder ein anderes Tool Ihrer persönlichen Präferenz) herunterladen und es über die Befehlszeile ausführen. Im Folgenden finden Sie die Schritte zum Herunterladen und Installieren von [!DNL Cygwin]:

   1. Navigieren Sie zu https://cygwin.com/install.html
   2. Laden Sie `setup-x86_64.exe` herunter und führen Sie es aus.

   <b>*Hinweis</b>:  Ihr Basisverzeichnis lautet: `C:\cygwin64\home\WINDOWSUSER`

   Sie können während der Installationsphase nach zusätzlichen Paketen suchen und diese installieren. Ich empfehle, alles zu installieren, was mit „`curl`“ und „`ssh.`“ zusammenhängt.*


   Hier finden Sie den auszuführenden Befehl:
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Hinweis&lt;/b>:  Ihnen werden mehrere Fragen gestellt, die Sie zum Generieren des Zertifikats ausfüllen müssen. Siehe Screenshot unten:

    Nachdem die Dateien erstellt wurden, konvertieren Sie „secret.key“ mithilfe des Befehls 
 zu „secret.pem“.<br> <br> <br>
